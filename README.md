# Salary Calc

Веб-калькулятор зарплаты, аванса, отпускных, премий и НДФЛ по прогрессивной шкале РФ. Проект полностью AI generated.

Публичный домен проекта: `salary.tupoybot.ru`.

## Что умеет

- считает таблицу выплат на выбранный год;
- принимает зарплату до вычета, число аванса и число зарплаты;
- переносит выплату на предыдущий рабочий день, если выбранная дата попала на выходной или праздник;
- считает аванс пропорционально рабочим дням с 1 по 15 число;
- не переносит зарплатный остаток декабря прошлого года в январь выбранного года;
- добавляет отпускные и уменьшает окладную часть за рабочие дни отпуска;
- добавляет больничные, уменьшает окладную часть за рабочие дни болезни и считает пособие по стажу;
- добавляет премии отдельными выплатами;
- применяет прогрессивную шкалу НДФЛ 13/15/18/20/22% и показывает ставку в колонке налога;
- сохраняет введённые данные только в `localStorage` браузера.

Сервис не хранит данные пользователя на сервере, не использует базу данных и не ведёт историю расчётов.

## Источники правил

- НДФЛ: новая пятиступенчатая шкала с 01.01.2025, ставки 13%, 15%, 18%, 20%, 22% по основной налоговой базе: https://www.nalog.gov.ru/rn69/ifns/imns69_03/info/15624246/
- Сроки зарплаты, перенос выплаты накануне выходного или праздника, отпускные не позднее чем за три дня: ст. 136 ТК РФ: https://www.consultant.ru/document/cons_doc_LAW_34683/a0a891ee650687026ef53d5d1194983419be6793/
- Расчёт аванса за первую половину месяца пропорционально отработанному времени: разъяснение Роструда от 30.01.2025: https://git55.rostrud.gov.ru/news/1285493.html
- Производственный календарь: isDayOff API: https://www.isdayoff.ru/docs/
- Упрощённая формула среднего дневного заработка для отпускных использует 29,3 календарного дня: постановление Правительства РФ N 922: https://normativ.kontur.ru/document?documentId=286435&moduleId=1
- Больничные: средний заработок за два предшествующих года / 730, коэффициент стажа 60/80/100%, НДФЛ удерживается независимо от источника выплаты: СФР https://sfr.gov.ru/employers/lk_insurer/baza_znanij/eln_vyplaty_oss_1/
- Максимальный средний дневной заработок для больничных в 2026 году: 6 827,40 ₽: буклет СФР https://sfr.gov.ru/files/id/press_center/pr/booklet/2026/Vyiplatyi_po_obyazatelnomu_sotsialnomu_strahovaniyu_2026.pdf

## Модель расчёта

Таблица строится по фактическим датам выплат внутри выбранного года. Декабрь прошлого года не добавляется в январь. Зарплата за декабрь текущего года попадёт в таблицу, только если после переноса следующей январской даты фактическая выплата оказалась в декабре текущего года. Отпускные и премии тоже попадают в таблицу по фактической дате выплаты.

В колонке НДФЛ показывается сумма налога, применённые ставки прогрессивной шкалы и эффективная ставка за строку месяца. Если месяц пересекает налоговый порог, ставка отображается как набор ставок, например `13%/15%`.

Аванс:

```text
оклад / рабочие дни месяца * рабочие дни с 1 по 15, не попавшие в отпуск
```

Зарплатная часть:

```text
оклад / рабочие дни месяца * рабочие дни с 16 числа до конца месяца, не попавшие в отпуск
```

Отпускные считаются приближённо:

```text
(оклад * 12 + премии в 12 месяцах до месяца отпуска) / 12 / 29,3 * дни отпуска
```

Нерабочие праздничные дни, которые isDayOff возвращает кодом `8`, не включаются в дни отпуска. Обычные выходные внутри отпуска включаются.

Больничные считаются как собственная болезнь или травма:

```text
min(max(доход за 2 предыдущих года / 730, минимум из МРОТ), максимум по базе взносов) * коэффициент стажа * календарные дни больничного
```

Коэффициент стажа выбирается в интерфейсе: 60% до 5 лет, 80% от 5 до 8 лет, 100% от 8 лет. Если доход за два предыдущих года не заполнен, используется приближение `оклад * 24`. Дата выплаты больничного моделируется как десятый рабочий день после окончания больничного.

## Запуск через Docker

```bash
docker build -t salary-calc .
docker rm -f salary-calc 2>/dev/null || true
docker run -d --name salary-calc --restart unless-stopped -p 8080:8080 salary-calc
```

После запуска откройте:

```text
http://localhost:8080
```

## Публикация Docker-образа

GitHub Actions собирает образ и публикует его в GHCR:

```text
ghcr.io/tupoybot/salary-calc:latest
```

## Локальный запуск без Docker

```bash
python3 -m venv .venv

pip install -r requirements-dev.txt
flask --app app.server run --host 0.0.0.0 --port 8080 --debug
```

## Ограничения

- расчёт отпускных приближённый и не заменяет бухгалтерский расчёт среднего заработка при неполных месяцах, больничных, командировках и особых выплатах;
- расчёт больничных покрывает базовый случай собственной болезни или травмы и не моделирует уход за ребёнком или членом семьи, стаж меньше 6 месяцев, замену расчётных лет, районные коэффициенты и особые основания снижения пособия;
- региональные производственные календари РФ не учитываются;
- налоговые вычеты, районные коэффициенты, материальная выгода и специальные налоговые базы не моделируются.
